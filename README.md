# STEP Weekly Practice Statements

This repository contains implementations of Weekly STEP Practice Problems.

---
# 01 Week 01 and 02 Practice Statement (Hash Table Fundamentals and Implementation)

This repository contains implementations of **10 real-world problems using Hash Tables** and related data structures.  
The problems are divided into **Week 1** and **Week 2**, demonstrating different applications such as caching systems, analytics dashboards, autocomplete systems, rate limiting, and financial transaction analysis.

# Week 1

Week 1 focuses on **basic to intermediate applications of hash tables** in practical systems.

**Problem 1 – Social Media Username Availability Checker**  
Checks if a username already exists using fast hash lookup and suggests alternative usernames.

**Problem 2 – E-commerce Flash Sale Inventory Manager**  
Manages product inventory during high-demand flash sales and prevents overselling.

**Problem 3 – DNS Cache with TTL**  
Implements a DNS caching mechanism with time-based expiration for domain-IP mappings.

**Problem 4 – Plagiarism Detection System**  
Detects similarity between documents using n-gram hashing techniques.

**Problem 5 – Real-Time Analytics Dashboard for Website Traffic**  
Processes website events to generate statistics like top pages and traffic sources.

---

# Week 2

Week 2 focuses on **advanced real-world systems built using hashing techniques**.

**Problem 6 – Distributed Rate Limiter for API Gateway**  
Controls API request limits per client using token bucket rate limiting.

**Problem 7 – Autocomplete System for Search Engine**  
Generates search suggestions based on prefix matching and query frequency.

**Problem 8 – Parking Lot Management with Open Addressing**  
Manages parking spot allocation using open addressing and linear probing.

**Problem 9 – Two-Sum Problem Variants for Financial Transactions**  
Detects suspicious financial transaction combinations such as two-sum, k-sum, and duplicates.

**Problem 10 – Multi-Level Cache System with Hash Tables**  
Simulates a caching architecture with multiple levels (memory, SSD, database).
